the agent enough context to calculate camera positions and timing without guesswork.

## Notes

- FRAME describes what the camera shows — which panels, how much of the scene, what's in/out of view
- ACTION describes what changes on screen — typing, form fills, content appearing, state changes
- FEEL describes the camera energy — is it a slow pan, a quick snap, a held shot?
- TREATMENT is optional — use for special effects like scrims, floating overlays, glowing borders
- Reference images help the agent understand the UI content for each beat — attach or link them
- Not every beat needs a reference image — only where the visual content matters
